1. Comprendre Mon PF App

Le projet est compose de deux applications: une API Laravel qui stocke les donnees, et une application Flutter qui affiche l'experience client, admin et livreur.

Frontend

mon_pfapp

Application Flutter. Elle contient les ecrans, le theme, les images de plats, la navigation, le panier et les appels HTTP.

Backend

mon_pfapi

API Laravel. Elle fournit les endpoints: inscription, login, menu, commandes, stats admin et livraison.

Flux principal

  1. Le client se connecte ou cree un compte.
  2. Il consulte le menu et ajoute des plats au panier.
  3. Flutter envoie la commande a Laravel.
  4. Le restaurant met a jour le statut.
  5. Le livreur accepte et suit la livraison.

3. Lancer l'API Laravel

Le backend doit rester ouvert pendant que Flutter tourne. Si tu fermes la fenetre Laravel, l'application mobile ne peut plus charger le menu ni envoyer les commandes.

Fenetre PowerShell 1: base + serveur

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapi"
..\tools\runtime\php-8.4.22\php.exe artisan migrate:fresh --seed
..\tools\runtime\php-8.4.22\php.exe artisan serve --host=0.0.0.0 --port=8000

Ne ferme pas cette fenetre. Le serveur doit afficher une URL du type http://0.0.0.0:8000.

Fenetre PowerShell 2: tester l'API

Invoke-RestMethod http://127.0.0.1:8000/api/health
Invoke-RestMethod http://127.0.0.1:8000/api/menu

Si /api/health retourne status: ok, le backend est pret.

4. Lancer sur Android, Windows ou Web

Choisis la cible. Le point important est toujours API_BASE_URL: Android reel utilise l'adresse IP du PC, Windows et Web utilisent souvent 127.0.0.1.

Trouver l'adresse IP du PC

ipconfig

Prends l'IPv4 de la carte Wi-Fi, par exemple 192.168.1.20.

Lancer sur telephone

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapp"
C:\Users\LOQ\dev\flutter\bin\flutter.bat devices
C:\Users\LOQ\dev\flutter\bin\flutter.bat run -d <device-id> --dart-define=DEMO_MODE=false --dart-define=API_BASE_URL=http://ADRESSE_IP_PC:8000/api

Remplace ADRESSE_IP_PC par l'IPv4 trouvee avec ipconfig.

Lancer sur emulateur Android

Dans l'emulateur Android, 10.0.2.2 signifie "mon PC".

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapp"
C:\Users\LOQ\dev\flutter\bin\flutter.bat run -d emulator --dart-define=DEMO_MODE=false --dart-define=API_BASE_URL=http://10.0.2.2:8000/api

Lancer sur Windows

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapp"
C:\Users\LOQ\dev\flutter\bin\flutter.bat run -d windows --dart-define=DEMO_MODE=false --dart-define=API_BASE_URL=http://127.0.0.1:8000/api

Si l'erreur parle de atlstr.h, installe le composant Visual Studio "C++ ATL".

Lancer sur Web Chrome

Le backend contient maintenant CORS pour permettre au navigateur d'appeler /api en local.

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapp"
C:\Users\LOQ\dev\flutter\bin\flutter.bat run -d chrome --web-port=5600 --dart-define=DEMO_MODE=false --dart-define=API_BASE_URL=http://127.0.0.1:8000/api

Mode demo sans backend

Utile si le jury veut voir l'interface rapidement ou si l'API est temporairement bloquee.

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapp"
C:\Users\LOQ\dev\flutter\bin\flutter.bat run -d chrome --dart-define=DEMO_MODE=true

6. Assistant debug pour Yassmine

Colle une erreur Flutter, Laravel, Android, Windows ou Web. Sans cle Gemini, l'assistant utilise une base locale de solutions. Avec une cle Gemini, il donne une reponse adaptee au contexte exact du projet.

La cle n'est jamais enregistree dans le repo. Le bouton "Enregistrer la cle" la garde uniquement dans ce navigateur, sur cette machine.

Pret. Colle une erreur, choisis la cible, puis clique sur Analyser.

7. Erreurs connues et solutions rapides

Ces cas sont les plus probables pendant l'installation ou la soutenance.

Android ne voit pas l'API

Cause probable: le telephone n'utilise pas 127.0.0.1. Il doit appeler l'IP Wi-Fi du PC.

ipconfig
flutter run -d <device-id> --dart-define=DEMO_MODE=false --dart-define=API_BASE_URL=http://ADRESSE_IP_PC:8000/api

Windows: atlstr.h introuvable

Cause probable: composant ATL absent dans Visual Studio Installer.

  1. Ouvrir Visual Studio Installer.
  2. Modifier Visual Studio Community.
  3. Installer "C++ ATL for latest v143/v14x build tools".
  4. Relancer flutter run -d windows.

Laravel: connection refused

Cause probable: artisan serve n'est pas lance ou le port 8000 est pris.

cd "C:\Users\LOQ\Documents\yassmin pfe\mon_pfapi"
..\tools\runtime\php-8.4.22\php.exe artisan serve --host=0.0.0.0 --port=8000

Web: blocked by CORS

Cause probable: le navigateur refuse une API sans headers CORS. Le projet ajoute HandleCors et config/cors.php. Relance Laravel apres changement.
